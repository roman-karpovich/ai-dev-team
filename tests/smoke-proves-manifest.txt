# smoke-proves-manifest.txt
#
# Purpose:
#   Maps smoke helper function names to the primary class of evidence they
#   provide. Helpers absent from this manifest are counted as unclassified in
#   the smoke summary.
#
# Format:
#   <helper_name> <class>
#
# Classes:
#   behavioral  - executes code, hooks, scripts, or fixtures and checks runtime behavior
#   schema      - validates structured file format or required structural keys
#   prompt-text - pins literal prose or prompt text
#
# Blank lines and lines starting with "#" are ignored.
#
# Probe G/H detector pins:
#   The probe-G/H detector and parser/boundary pins run the probe and compare
#   its stdout against a frozen expected_stdout.json with the volatile
#   receipt_metadata.trigger_input_hash stripped from both sides (a
#   hash-stripped canonical compare, NOT a raw byte-diff) plus a structural
#   gate against probe-envelope.schema.json. They genuinely assert
#   detection/parse logic, so they are classified `behavioral`. The separate
#   probe-*-fixtures-schema-conform pins validate only the fixture envelope
#   shape and are classified `schema`.
#
# Runtime:
#   The canonical smoke harness loads this manifest into a Bash 4+ associative
#   array when available (`declare -A PROVES_MAP`) and falls back to parallel
#   indexed arrays on Bash 3.2 (stock macOS) — see the PROVES_MAP_SUPPORTS_ASSOC
#   branch in tests/smoke.sh.
